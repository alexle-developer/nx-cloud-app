Overview

The primary purpose of the Nx-Cloud-App application is to demonstrate the power of leveraging Nx Monorepo with Angular NgRx state management. Therefore, we are focusing on something other than building components and structures for the Nx-Cloud-App.

To demonstrate NgRx, our sample application must be complex enough to have state management requirements but simple enough to implement those requirements within a reasonable amount of time. Therefore, we start with a simple Angular reactive form "Products" application that does not use NgRx.

Our intention is to replace the state management and component interactions with NgRx for Products NgRx.

Sample Application

  • On the left panel, a list of products is displayed.
  • The "Display Product Code" checkbox is to toggle product code in the list on and off.
  • Clicking on a product name displays the product details page.
  • The sample application has a root application module called App Module.
  • The application's features are defined in the producs library.
